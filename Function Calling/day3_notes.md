# Day 3: Function Calling 学习笔记

## 我做了什么
- 实现了单工具调用
- 实现了多工具自主选择
- 实现了循环式 Agent

## 反直觉的发现
1. Agent在对话过程不只是LLM的传声筒，会使LLM更聪明
2. Agent 的本质不是给 LLM 提供数据,而是让 LLM 突破'纯文本生成'的边界,通过工具与外部世界交互

## 工具描述写法的心得
工具用途，调用时机，参数格式，边界情况

## Agent 循环的核心机制
LLM不在调用agent的工具说明LLM已经解答完成，另外需要限制单次问题循环最大次数，防止过多请求或者陷于死循环
